## 为什么旧课程不能直接发布

早期课程虽然满足词汇白名单，但出现过下列机械生成问题：

- `This is Friday.` 之外的日历词与普通名词没有区分；
- 情态词、代词、介词和连词被错误套入普通名词句型；
- `This is cannot.` 一类词性错误；
- `This is -ant.` 一类词典词缀污染；
- `an unit` 一类只按首字母判断冠词的错误；
- `I can happen.` 一类主语语义不自然的动词模板；
- 词典先列罕见名词义，导致 `This is an old.` 一类核心词义偏差。

这些问题说明“零生词”只能保证词汇范围，不能单独保证教材质量。因此最终发布链增加了独立内容质量门禁。

## 复核与修复层级

### 1. 严格零生词校验

每个单元允许出现的英文词仅包括：

- 之前课程已经正式教授的词；
- 当前单元正式教授的新词。

校验覆盖：词组、例句、句子练习、阅读标题、阅读正文和英文答案选项。多词条目按完整词序列检查。

### 2. 数据污染清理

课程最终化会清理词典中的词缀、缩写和显示形式污染，并对已确认的词条修正音标和核心释义。

### 3. 261 条审核句型

对代词、介词、连词、限定词、数量词、不规则复数、只能作表语的形容词、需要人物宾语的动词和高风险副词建立了数据驱动的审核句型。

例如：

- `My book is here.`，而不是把 `my` 当普通名词；
- `The police are here.`，使用正确的集合名词一致关系；
- `I can persuade him.`，而不是 `persuade this`；
- `I am awake.`，而不是 `This is awake.`；
- `This is a unit.`，根据发音使用 `a` 而不是 `an`。

### 4. 71 条核心词义修正

对多词性词条进行常用核心义复核，同时修改中文释义和英文上下文。

例如：

- `old` 使用常用形容词义“老的；旧的”；
- `own` 使用动词义“拥有”；
- `most` 使用限定词义“大多数”；
- `sick` 使用形容词义“生病的”；
- `entire` 使用形容词义“整个的”；
- `surrounding` 使用形容词义“周围的”。

### 5. 通用语义规则

对可以安全归类的词使用受控语义模板：

- 抽象或不可数名词：`This is about information.` / `I know about information.`；
- 事件类动词：`It can happen.`；
- 基数词：`I have four books.`；
- 序数词：`This is the third.`；
- 日历词：`It is Friday.`。

### 6. 课程前置词补充

为了保证自然句型仍然零生词，基础阶段正式加入了必要工具词和词形，而不是设置隐藏豁免：

- 常用复数形式；
- `the / an / have / these / those / word`；
- `about / know / use / make / good / thing`；
- `was / more / than / ago / recently / previously`。

## 发布门禁

完整课程必须依次通过：

1. 课程结构和重复词条检查；
2. 累计词汇白名单检查；
3. 每个新词至少两个阅读上下文；
4. 例句与答案选项唯一性检查；
5. 确定性内容质量检查；
6. AI 辅助全量语义复核。

任意一层失败，课程文件和 Windows 便携包均不应作为正式版本发布。

## 本轮审计方式

本轮由 GPT-5.5 Thinking 对课程生成模式、异常词类和多词性核心义进行审阅，并将审阅结论固化为可重复执行的数据表和 Rust 规则。随后使用与 Rust 最终化优先级一致的本地审计程序，对全部 3083 个词条和 518 个单元进行穷举验证，而不是只做随机抽样。
